# V1.4 Scope

V1.4 is the smallest useful upgrade from the current V1.35 single-file patch MVP.

## Goal

Make the agent more reliable when the model output is malformed, fenced, no-op, or otherwise low quality, without changing the current single-target design.

## In Scope

- Local, rule-based gatekeeping before patch application
- One retry on patch-generation or patch-parse failure
- Quality checks that reject no-op or clearly malformed patches
- Strong failure cleanup that restores the repo to `HEAD`
- Clear failure reasons written to `workspace/`
- Single target file only

## Out of Scope

- Multi-file edits
- Planner agents
- Semantic search
- RAG / long-term memory
- Automatic commit or push
- Self-directed multi-step debugging loops

## Proposed Flow

Task
↓
Context Builder
↓
Builder
↓
Gatekeeper
↓
Patch Applier
↓
Tests
↓
Retry once on failure
↓
Rollback or success

## Success Criteria

- Bad patch output is rejected before it can damage the target file
- A valid patch produces a real diff
- Failures always leave the repo clean
- The latest result is always visible in `workspace/` and `TEST_STATUS.md`
