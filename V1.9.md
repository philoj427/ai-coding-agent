# V1.9 Plan-only Mode

## Goal

V1.9 adds a safe stop point for tasks that are too risky for the single-file patch workflow.

The planner may still identify the likely target, related tests, and reason, but high-risk tasks do not enter patch generation.

## Behavior

```text
natural-language task
project_index.json
Planner
Plan validator
low / medium risk -> existing single-file patch workflow
high risk -> result.json with status=plan_only
unclear target -> result.json with status=plan_failed
```

## Plan-only Cases

- Protected files such as `ai_coding_agent/workflow.py`
- Core safety files such as `ai_coding_agent/git_guard.py`
- Requests that imply multi-file edits
- Requests that require manual review before code changes

## Outputs

High-risk task:

```json
{
  "status": "plan_only",
  "reason": "Task targets protected or high-risk file ai_coding_agent/workflow.py.",
  "target_file": "ai_coding_agent/workflow.py",
  "test_type": "none",
  "test_file": null
}
```

Unclear task:

```json
{
  "status": "plan_failed",
  "reason": "..."
}
```

## Safety Boundary

- Still one target file only.
- High-risk plans never patch code.
- Unknown or ambiguous tasks fail closed.
- Existing Gatekeeper, tests, rollback, and Git Guard remain unchanged for patchable tasks.
