# V1.7 Task Planner

## Goal

V1.7 adds a planner step before the existing single-file patch workflow.

The user can provide a plain-language task. The planner produces `task_plan.json`, then the existing safety pipeline handles candidate selection, patch generation, Gatekeeper checks, syntax validation, tests, and Git Guard.

## Flow

```text
User task
Planner
task_plan.json
Plan validator
Candidate selection
Replacement generation
Gatekeeper
Apply
py_compile
Tests
Git Guard
```

## Task Plan

The planner emits JSON with:

```json
{
  "target_file": "demo_add.py",
  "test_type": "unittest",
  "test_file": "tests/test_demo_add.py",
  "risk_level": "low",
  "reason": "The request refers to add(), which is implemented in demo_add.py.",
  "allowed_files": ["demo_add.py"],
  "forbidden_files": ["ai_coding_agent/", ".gitignore", "README.md"]
}
```

## Safety Rules

- `target_file` must exist.
- `allowed_files` must contain exactly one file.
- `target_file` must be listed in `allowed_files`.
- `test_type` must be supported.
- `risk_level` must be `low` or `medium`.
- V1.7 still modifies only one target file.
