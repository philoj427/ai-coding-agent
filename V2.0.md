# V2.0 Safe Multi-step Change Plans

## Goal

V2.0 adds a safe multi-file shell around the existing V1.x single-file workflow.

It does not let the model patch multiple files at once. A multi-file task is split into single-file steps, and each step still uses the existing Gatekeeper, strict SEARCH/REPLACE patching, `py_compile`, tests, rollback, and Git Guard.

## Flow

```text
change_plan.json
Plan Validator
Step Runner
S1 single-file workflow
S2 single-file workflow
S3 single-file workflow
Final full test
Plan-level Git Guard
result.json
```

## Change Plan Format

```json
{
  "task": "Add divide zero guard and tests",
  "steps": [
    {
      "step_id": "S1",
      "target_file": "math_tool.py",
      "intent": "Add a zero-division guard to divide().",
      "allowed_change": "production_code",
      "test_type": "unittest",
      "test_file": "tests/test_math_tool.py"
    },
    {
      "step_id": "S2",
      "target_file": "tests/test_math_tool.py",
      "intent": "Add a test for divide by zero.",
      "allowed_change": "test_code",
      "test_type": "unittest",
      "test_file": "tests/test_math_tool.py"
    }
  ],
  "final_test_type": "unittest",
  "final_test_file": "tests/test_math_tool.py"
}
```

## Safety Limits

- Maximum 3 steps.
- Maximum 3 changed files.
- Protected files are rejected.
- Each step targets exactly one file.
- `production_code` steps cannot target test files.
- `test_code` steps must target test files.
- Any step failure rolls back the whole plan to `HEAD`.
- Final test failure rolls back the whole plan to `HEAD`.
- Plan-level Git Guard only allows files listed in the change plan.

## Usage

```powershell
python .\agent.py --root . --change-plan workspace\change_plan.json --model qwen2.5-coder:7b
```

## Verification

- `python -m unittest discover -s tests`: 53 tests passed.
- V2.0 unit coverage includes a real two-step source + test modification against `math_tool.py` and `tests/test_math_tool.py`.
- The V2.0 runner cleans final-test `__pycache__` files before plan-level Git Guard.

## Boundary

V2.0 is not a broad autonomous refactor system. It is a controlled multi-step executor for small source + test changes.
