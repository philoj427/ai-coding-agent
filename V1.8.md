# V1.8 Project Index

## Goal

V1.8 adds a readable project index so the planner can choose a target file and test file from repo facts instead of guessing from filenames in the prompt.

## What It Builds

- `workspace/project_index.json`
- indexed source files
- Python symbols from AST
- simple JavaScript symbols from text rules
- test mappings such as `demo_add.py` to `tests/test_demo_add.py`
- protected files from `memory/PROTECTED_FILES.md`

## Planner Behavior

The planner now uses the project index first:

```text
natural-language task
project_index.json
protected files
rule-based plan
validated task_plan.json
single-file patch workflow
```

If the index cannot identify a clear target, the workflow fails closed unless Ollama returns a valid plan that passes validation.

## Safety Boundary

- V1.8 still supports exactly one target file.
- Protected files are rejected by the planner and validator.
- `allowed_files` must contain exactly one file.
- The existing Gatekeeper, `py_compile`, test runner, rollback, and Git Guard still run.
